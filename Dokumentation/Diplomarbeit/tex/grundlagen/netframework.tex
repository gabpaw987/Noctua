\subsection{Allgemein}

.NET ist ein Framework der Firma Microsoft, dass eine Vielfalt an Sprachbibliotheken für eine große Palette an Programmiersprachen zur Verfügung stellt. Früher arbeiteten alle Programmiersprachen von Microsoft, wie bspw. C++, mit der WinAPI-32. Nun wurde diese \gls{api} mit .NET durch ein sprachenunabhängiges, weitaus umfangreicheres Framework ersetzt. \cite{visualcsharp}\\
\\
Insgesamt wird die Nutzung des .NET-Frameworks von über 30 Programmiersprachen unterstützt. die Richtlinien die eine Sprache einhalten muss, um als .NET-konform angesehen werden zu können sind von Microsoft als \gls{cls} definiert worden. Zu den berühmtesten Vertretern solcher Sprachen Zählen C\#, F\#, Visual Basic .NET (VB.NET) und C++. \cite{visualcsharp}\\
\\
Zu den wichtigsten Features des .NET-Frameworks zählen die folgenden:
\begin{itemize}
\item \textbf{Objektorientierung} \\
	Das .NET-Framework ist zu 100\% objektbasiert. Das bedeutet jegliche Elemente, sogar einfache Datentypen wie bspw. Integer, lassen sich auf Objekte zurückführen. Ja sogar die internen Zugriffe des Frameworks auf das darunterliegende Betriebssystem sind in Klassen gekapselt.
\item \textbf{Plattformunabhängigkeit} \\
	Anwendungen, die das .NET-Framework benutzen, werden ähnlich der \gls{jvm} erst zur Laufzeit in Maschinencode umgewandelt. Weiters ist die Spezifikation der von Microsoft benutzten Laufzeitumgebung, der sog. \gls{clr}, offen zugänglich. Dadurch verlangt die Nutzung von .NET-Anwendungen also eine solche Umgebung, diese kann allerdings auch auf Plattformen portiert werden, die nicht Windows heißen. So gibt es neben der proprietären Windows-Implementierung des .NET-Frameworks von Microsoft bspw. auch das \textit{Mono}-Projekt, mit dem .NET bereits erfolgreich auf Linux portiert werden konnte.
\item \textbf{Sprachenunabhängigkeit} \\
	Alle Komponenten des .NET-Frameworks können von jeder unterstützten Sprache problemlos verwendet werden. Das .NET-Framwork ermöglicht aber auch die Nutzung jeglicher, in einer .NET-konformen Programmiersprache geschriebenen Komponenten, in jeder anderen .NET-konformen Programmiersprache. So können zum Beispiel alle in C\# 2010 geschriebenen Klassen auch in F\# oder VB.NET genutzt und sogar abgeleitet werden.
\item \textbf{Speicherverwaltung} \\
	Die explizite Freigabe von nicht mehr benötigtem Speicher hat in der Vergangenheit schon zu vielen Problemen geführt. Dafür wurde mit dem .NET-Framework ein \textit{Garbage Collector} eingeführt, der dem Programmierer diese Aufgabe automatisch abnimmt. 
\item \textbf{Weitergabe} \\
	Auch die Weitergabe konnte deutlich vereinfacht werden. So können auf .NET basierende Sprachen einfach in eine .EXE- oder .\gls{dll}-Datei kompiliert und von jedem beliebigen Ordner aus ausgeführt werden. Eine .EXE-Datei ist dabei eine direkt ausführbare Datei, deren Code bereits in einzelne Bytes umgewandelt wurde. \gls{dll}-Bibliotheken werden im folgenden Punkt erläutert. \cite{visualcsharp}
\end{itemize}
